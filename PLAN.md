# ⚾ Angry Baseball — 憤怒棒球 企劃 & 技術架構

第一人稱打擊視角的 3D 棒球遊戲。投手把球投過來,玩家抓時機揮棒,依飛行距離與全壘打計分,含全球排行榜與球棒商店。
**技術完全沿用 angry-pig 驗證過的架構**:three.js r160 渲染 + cannon-es 物理 + Vite 建置 + Cloudflare Pages Functions + D1 後端。

---

## 1. 核心玩法(Core Loop)

```
投手準備動作 → 投球(球飛向本壘) → 玩家瞄準 + 抓時機點擊揮棒
→ 判定擊球品質 → 球飛出去(物理 + 慢動作球視角) → 落點量距離
→ 跳分數/Combo → 下一球
```

### 1.1 揮棒判定(手感核心,好玩的關鍵)

不做真實球棒剛體碰撞(不穩定、難調手感),改用 **「時機 + 準星」混合判定**,再把結果轉成物理衝量交給 cannon-es:

- **時機軸(左右方向)**:球通過本壘板有一個「理想觸擊時刻」。
  - 提早揮 → 拉打(往左外野)、太早 → 界外
  - 準時揮 → 強襲中外野(**PERFECT!** 判定,金色跳字 + Hit-stop 頓幀)
  - 晚揮 → 推打(往右外野)、太晚 → 揮空
- **準星軸(高低仰角)**:滑鼠準星相對球的位置決定發射仰角。
  - 打球心下緣 → 高飛球(距離遠但可能被接殺區抓到)
  - 打正中 → 平飛強襲球
  - 打上緣 → 滾地球(低分)
- **擊球品質分級**:`揮空 / 界外 / 普通 / 好球 / 完美`,對應出速倍率 0 / 0.4 / 0.7 / 0.9 / 1.15。
- 完美擊中:Hit-stop 0.08s + 畫面微震 + 白閃 + 「鏗!」金屬音 —— 全部是 angry-pig 已有的 juice 技術。

### 1.2 球的物理(cannon-es)

- 投球:球體剛體從投手丘(約 18m 外)以初速 + 自旋參數飛來。
- **Magnus 力**自己加(每 tick `body.applyForce(spin × velocity 的簡化式)`),做出變化球:
  | 球種 | 行為 |
  |------|------|
  | 直球 | 快、直 |
  | 曲球 | 慢、大幅下墜 |
  | 滑球 | 中速、橫向位移 |
  | 蝴蝶球 | 慢、隨機飄(perlin 抖動) |
  | 火球(Boss 投手) | 極快 + 拖尾特效 |
- 擊出後:依判定結果給衝量,linearDamping 模擬空氣阻力,落地第一觸點量測距離。
- 打擊瞬間可以直接沿用 angry-pig 的拋物線預測程式碼反過來畫「預測落點」(高級球棒的特殊能力)。

### 1.3 計分

```
單球得分 = 距離(公尺) × 10
         + 全壘打加成(牆外 +2000,場地標示區加倍:120m 深遠中外野 ×2)
         + PERFECT 時機加成 +500
         + 場上目標加成(見 1.4)
         × Combo 倍率(連續安打 ×1.1 → ×1.2 → … 上限 ×2,揮空歸零)
```

- 落點即時跳字(沿用 floaters 系統):「**128m!全壘打!**」「騰空!」「爆遠!」
- 距離里程碑跳字沿用 angry-pig 的 `MS_DIST` 機制。

### 1.4 外野目標(把 angry-pig 的破壞玩法搬進球場 —— 好玩的秘密武器)

外野與觀眾席擺**可破壞物件**,直接複用現有 glb 資產與剛體堆疊技術:

- 🪧 **廣告看板**(BrickWall / WoodPlanks 堆的):打穿 +800
- 🛢️ **爆炸桶塔**(ExplodingBarrel):打中連環爆,炸飛周圍動物觀眾 +1500
- 🐷 **動物觀眾**(Pig/Sheep/Chicken… 站在外野看台上):球砸中 → 撞飛 + 賞金(彩蛋:跟 angry-pig 世界觀連動,「豬的復仇」)
- 🚌 **移動目標**:一台車/飛船緩慢橫越深遠外野,打中 JACKPOT +5000
- 🎯 **空中連擊**:全壘打球在空中撞到飛過的雞 → 「空中安打!」+1000(致敬 angry-pig 的空中狙擊)

### 1.5 球視角(Ball-Cam,爽度核心)

擊球品質 ≥ 好球時,鏡頭切到**追球慢動作**(0.3× 時間 1 秒 → 恢復),跟著球飛越球場、看它砸進看板或飛出牆外 + 煙火。這是本作最重要的「爽」來源,優先做。

## 2. 遊戲模式

| 模式 | 規則 | 排行榜 |
|------|------|--------|
| 🏆 **全壘打大賽** | 10 球定勝負,總距離+加成為分數 | ✅ 主榜 |
| 🔥 **生存連打** | 球速/球種越來越刁鑽,揮空 3 次淘汰,算連打數與總分 | ✅ |
| 📖 **故事挑戰** | 15 關,對戰不同投手(小豬→狼→Boss 馬投手),過關拿星星解鎖,依表現 1~3 星 | — |
| 🚀 **打飛最遠** | 單球最遠距離紀錄 | ✅(沿用 angry-pig「飛最高」榜的做法) |

- 生存模式每 5 波出 **Boss 投手**(放大紅色調,火球 + 假動作),擊出 Boss 的球得分 ×3 —— 複用 angry-pig Boss 系統。
- 4 種 biome 場景直接沿用:🌿 白天球場 / 🌅 黃昏 / ❄️ 雪地 / 🌃 夜間燈光球場(夜場全壘打有煙火,最漂亮)。

## 4. 操作

| 平台 | 操作 |
|------|------|
| 桌機 | 滑鼠移動瞄準(準星)、**左鍵點擊揮棒**、R 要求重投(每局限 1 次)、M 音樂、Esc 暫停 |
| 手機 | 沿用 angry-pig 虛擬搖桿:左下控準星、右下大顆「揮棒」鈕 |

不需要蓄力(節奏遊戲抓時機比蓄力適合棒球),所以比 angry-pig 的操作更簡單直覺。

## 5. 技術架構 & 檔案結構

```
angry-baseball/
├── index.html          # HUD + 登入 + 選單 + 商店/排行榜/留言板彈窗(抄 angry-pig 版型改皮)
├── game.js             # 主程式(場景、物理、投打判定、模式流程)— 對應 fps.js
├── pitch.js            # 投手 AI:球種、配球邏輯、Magnus 力、投球動畫
├── batting.js          # 揮棒判定、擊球品質、衝量計算、Ball-Cam
├── field.js            # 球場搭建:內外野、全壘打牆、看板、目標物、biome
├── sfx.js              # Web Audio 合成:球棒「鏗」、捕手手套「啪」、觀眾歡呼(音量隨距離漸強)
├── functions/api/      # 直接複製 angry-pig 的後端(score/leaderboard/heartbeat/online/messages/totals/_lib)
├── schema.sql          # 同 angry-pig 結構,mode 欄位改棒球模式
├── wrangler.jsonc      # 新 D1:angry-baseball-db
├── public/
│   ├── _headers        # CSP 安全標頭(照抄)
│   └── assets/         # 動物 glb 照搬 + 新增:球棒模型(可先用程式生成圓柱)、球(球體+縫線貼圖)
└── vite.config.js
```

### 新開發重點(真正要寫的部分)

1. **投打時機判定**(batting.js)— 手感調校是成敗關鍵,參數全部拉成常數方便調
2. **Magnus 變化球**(pitch.js)
3. **球場幾何**(field.js)— 扇形草皮貼圖(改 makeGrassTexture 加內野土)、全壘打牆、距離標示
4. **Ball-Cam 慢動作追球**
5. **球棒商店數據 + 特效 hook**
6. 投手用現有 Pig 模型 + 投擲動作(沒有投球動畫就用程式擺 pose:抬手→甩臂)

## 6. 後端(Cloudflare Pages Functions + D1)

- `POST /api/score`:帶 `mode`(derby/survival/farthest)與輔助數據(全壘打數/最遠距離),沿用 IP 限流 + 合理性檢查(例:單球距離 > 250m 直接擋)
- `GET /api/leaderboard?mode=`:三張榜
- heartbeat / online / messages / totals 全部照搬
- 開發時無 `/api` 自動回退 localStorage(angry-pig 已有這機制)

## 8. 好玩度設計原則(從 angry-pig 學到的)

1. **每一球都要有回饋**:跳字、音效、震動,絕不無聲無息。
2. **失敗也要好笑**:揮空有「咻——」音效 + 捕手接球「啪」,鹹魚棒之類的搞笑內容。
3. **偶爾的大獎時刻**:JACKPOT 移動目標、空中連擊、Boss 球 ×3 —— 間歇性強化最上癮。
4. **排行榜壓力**:首頁直接顯示「差 XXX 分就能上第 10 名」。
5. **短局制**:全壘打大賽 10 球約 90 秒一局,適合「再一局」循環與手機片段時間。
